# Test specs

This folder exists so you can try **Spec-together** against real Markdown files
without needing another repo.

Open the app, point it at this repository, and open any file under
`test-specs/`. Select a line (or Shift-click a range) and leave a comment — a
review-only pull request will be created to hold the conversation, and your real
branch stays untouched.

Files here:

- `product-overview.md` — what the app is and who it's for
- `comment-data-model.md` — how comments map onto GitHub primitives
- `ui-guidelines.md` — interaction and visual rules for the reviewer
- `open-questions.md` — deliberately unresolved decisions worth commenting on

These are intentionally opinionated and a little incomplete — good bait for
review comments.
